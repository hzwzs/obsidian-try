## 1. 这是什么？

**Reasonix Code** 是一个搭载在 Reasonix 平台上的 AI 编程助手。你通过对话的方式描述需求，它能够：

- **阅读并理解**你的代码库（任意规模）
- **搜索**代码中的符号、引用、定义
- **规划并执行**多步骤的代码修改（重构、修 Bug、加功能）
- **运行命令**（编译、测试、lint、启动 dev server）
- **查阅网络**文档或技术方案
- **记住**你的偏好和项目决策（跨会话持久化）

### 哲学

| 原则 | 说明 |
|------|------|
| **言行一致** | 每句关于代码的断言都必须有文件引用支撑，拒绝空口断言 |
| **成本意识** | 能一次搞定就不反复，能用小模型就不上大模型 |
| **安全为先** | 写操作有审核门（review / auto 两种模式），无意破坏极易回退 |
| **主动规划** | 大规模改动前提交计划供你审批，而不是闷头直接改 |

## 2. 快速上手

### 启动

```bash
cd your-project
reasonix code
```

这会在当前目录启动一个 Reasonix Code 会话，工作区锁定在此目录。

### 第一个对话

**问：*"这个项目是干什么的？"***

它会自动探索你的项目结构，给你一个概括。

**问：*"在 src/utils.ts 里加一个 `formatDate` 函数"***

它会读取目标文件、理解上下文、编写代码并以 SEARCH/REPLACE 块展示修改。

**问：*"帮我重构这个模块"***

它会先提交一份计划（Plan）让你审批，获批后再逐步骤执行。

## 3. 四大工作模式

### ① 自由对话模式（默认）

你问它答，需要时主动使用工具（读文件、搜索、执行命令）。适合日常问答、小修小改、探索性工作。

### ② 审核模式（Review Mode）

每个文件修改在落地前都会暂停，等你逐块审核批准。你可以在 TUI 中看到 **Apply / Reject** 按钮。  
适合代码审查严格、不希望自动写入的场景。

> 你**不需要手动切换**——编辑器会根据上下文自动选择 review / auto。你只需正常对话即可。

### ③ 计划模式（Plan Mode）

通过 `/plan` 进入。此模式下：

- **写操作工具被禁用**（编辑文件、创建文件、运行危险的 shell 命令）
- 只能读文件、搜索代码、查看目录
- 必须先调用 `submit_plan` 提交计划，你审批后才能继续执行

适合：架构变更、跨文件重构、有风险的清理操作。

### ④ 自动模式（Auto Mode）

无需审批，SEARCH/REPLACE 块直接落地到磁盘，但有撤销窗口可回退。  
适合：你完全信任当前改动、快速迭代的场景。

> 三种模式（review/auto/plan）由编辑器管理，你只需照常对话。

## 4. 会话指令（Slash Commands）

在对话输入框中以 `/` 开头触发的快捷指令：

| 命令 | 作用 |
|------|------|
| `/plan` | 进入计划模式（只读 + 提交计划） |
| `/skill <name>` | 直接调用某个已安装的技能（Skill） |
| `/search-engine <engine>` | 切换网络搜索后端：mojeek / searxng / metaso / tavily |
| `/checkpoint` | 手动创建快照（如果平台支持） |
| `/undo` | 撤销上一次操作（如果平台支持） |
| `/new` | 清空当前会话历史，重新开始 |

## 5. 工具箱速查

所有工具均由 Reasonix Code 在后台调用，你无需手动操作。了解它们有助于你理解它能做什么、不能做什么。

### 文件读取

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `read_file` | 读文件内容（支持指定行范围） | 查看某个函数的实现 |
| `get_file_info` | 查看文件类型、大小、修改时间 | 确认文件是否存在、是否新改动 |
| `glob` | 按 glob 模式搜索文件，按修改时间排序 | "最近改了哪些文件？"、"找出所有 test 文件" |

### 目录浏览

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `list_directory` | 列出目录下一级条目 | 快速查看项目根目录有什么 |
| `directory_tree` | 递归列出目录树（默认深度 2） | 了解模块结构 |
| `search_files` | 按文件名模糊搜索 | "WebSocket 相关的文件在哪？" |

### 代码搜索

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `search_content` | 按内容全文搜索（grep） | "哪里调用了 `connect()`？" |
| `search_files` | 按文件名搜索 | "所有 .config.ts 文件在哪？" |

### 文件编辑

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `edit_file` | SEARCH/REPLACE 精准替换 | 修改函数体、修复 Bug |
| `multi_edit` | 一次调用修改多个文件 | 跨文件重构、重命名符号 |
| `write_file` | 创建新文件或覆盖 | 新建模块、写入配置文件 |
| `move_file` / `copy_file` | 移动/复制文件或目录 | 重命名、重组文件结构 |
| `delete_file` / `delete_directory` | 删除文件/目录 | 清理废弃代码 |

### 命令执行

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `run_command` | 前台运行命令（60s 超时） | 测试、编译、lint、git 操作 |
| `run_background` | 后台长期运行进程 | Dev server、大文件下载、耗时构建 |
| `job_output` | 查看后台进程日志 | 检查 dev server 启动了没有 |
| `wait_for_job` | 等待后台进程结束 | 等下载完成、等构建结束 |
| `stop_job` / `list_jobs` | 管理后台进程 | 停掉不再需要的进程 |

### 规划与决策

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `submit_plan` | 提交多步骤执行计划供审批 | 大规模重构、跨文件修改 |
| `mark_step_complete` | 标记计划中的一步已完成 | 跟踪计划进度 |
| `revise_plan` | 修改剩余计划步骤 | 你的反馈改变了执行路线 |
| `ask_choice` | 以选项列表的方式让你做选择 | "用 A 方案还是 B 方案？" |

### 记忆与知识

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `remember` | 保存信息到持久记忆 | 你的工具链偏好、项目关键决策 |
| `forget` | 删除已保存的记忆 | 主观上不再需要的记忆 |
| `recall_memory` | 读取记忆详情 | 需要回顾之前记住的完整内容 |
| `web_search` | 搜索公开网络 | 查最新文档、版本号、技术方案 |
| `web_fetch` | 获取网页正文 | 阅读在线文档、API 规范 |

### 技能系统

| 工具 | 用途 | 适用场景 |
|------|------|----------|
| `run_skill` | 调用已安装的技能 | 执行预定义的复杂工作流 |
| `install_skill` / `create_skill` | 创建新技能 | 把你常用的多步骤任务录成技能 |
| `/skill <name>` | 同 run_skill，通过斜杠命令触发 | 快速调用 |

## 6. 典型工作流

### 工作流 A：探索一个未知项目

```
你: "这个项目是做什么的？"
助手: → directory_tree 看结构
      → read_file 读 README / package.json / entry point
      → 输出项目概述
```

### 工作流 B：定位并修复一个 Bug

```
你: "登录功能报错：'TokenExpiredError'"
助手: → search_content("TokenExpiredError") 定位抛出点
      → read_file 查看相关代码上下文
      → 分析根因（过期检查逻辑缺陷）
      → 提交 SEARCH/REPLACE 修复
```

### 工作流 C：增加新功能

```
你: "给 CLI 加一个 --verbose 参数"
助手: → read_file 查看 CLI 入口
      → read_file 查看参数解析方式
      → 规划改动路径（解析器 → 日志器 → 调用链）
      → 提交 multi_edit 修改所有涉及文件
      → 运行测试验证
```

### 工作流 D：复杂跨文件重构

```
你: "把 utils 里的所有日期函数拆成独立的 date.ts"
助手: → search_content 找出所有日期相关函数
      → 提交 Plan（列出每个文件要改什么）
      → 你审批 → 按步骤执行 → 每完成一步标记进度
      → 运行测试确认不破坏现有功能
```

### 工作流 E：让助手持续运行 Dev Server

```
你: "启动开发服务器，然后告诉我是否成功了"
助手: → run_background("npm run dev")
      → 等待启动信号（"Local:" / "listening on"）
      → 报告 URL 和端口
      → 你之后可随时要求查看日志或停掉它
```

## 7. 进阶技巧

### 🧠 让搜索更精准

```diff
- "搜索一下连接超时的问题"
+ "搜索代码中所有设置了 timeout 的 HTTP 请求在哪里"
```

具体的描述能让它直奔目标，而非遍历整个代码库。

### 📋 使用计划模式做手术级改动

当你觉得"改动有点大"时，主动说：

> "先做个计划"

它会调用 `submit_plan` 提交一个带步骤列表的计划，你可以逐项审阅再开动。

### 💾 利用记忆功能减少重复说明

每次启动新会话时，助手会加载之前保存的记忆。你可以说：

> "记住我偏好 pnpm 而不是 npm"
> "记住这个项目的测试命令是 pnpm test:ci"

下次会话它自动知道。

### 🎯 多工具并行

Reasonix Code 可以**并行调用多个工具**。比如同时读两个文件、同时搜索多个符号——这会大幅缩短等待时间。你只需一句话描述需求，它自己会安排。

### 🔄 从 Bug 到验证的闭环

让它修完 Bug 后，你可以直接说：

> "现在跑一下测试"

它会执行 `run_command("pnpm test")` 并把结果告诉你。红绿直接可见。

### 📦 创建技能录制常用操作

如果你每周都要做"更新 API 文档"一样的步骤，可以说：

> "把我上面做的那套流程录成一个叫 update-docs 的技能"

它就会创建一个可复用的技能文件，以后直接 `/skill update-docs` 搞定。

## 8. 文件操作约定

| 规则 | 说明 |
|------|------|
| **SEARCH 必须精确** | 编辑文件时 `SEARCH` 块必须逐字匹配现有内容，不会模糊匹配 |
| **路径相对于工作区** | 所有文件路径都以项目根目录为基准 |
| **先读后改** | 修改前一定会先读取文件的最新版本，确保 SEARCH 匹配 |
| **拒绝重复** | 同一个 SEARCH 模式在文件中出现多次时会拒绝修改（避免改错地方） |
| **原子批量** | `multi_edit` 在任一个修改失败时会回滚全部，不留下半残状态 |
| **不主动改依赖** | `node_modules`、`.git`、`dist`、`build` 等目录默认跳过 |

### 支持的路径写法

```text
src/utils.ts              ← 相对路径（推荐）
/src/utils.ts             ← POSIX 绝对（/ 代表项目根）
F:\project\src\utils.ts   ← Windows 绝对路径
```

> **注意**: shell 命令（`run_command`）中不要使用前导 `/`——POSIX 会把 `/src` 解析为文件系统根目录，而非项目根目录。命令中用相对路径即可。

## 9. 常见问题

### Q: 它会自己乱改代码吗？

**不会。** 默认模式下，每次 `edit_file` / `write_file` 调用都要经过审核（review mode）或处于你的可见范围内（auto mode 有撤销窗口）。计划模式更严格——写操作完全禁用，必须你批准计划后才能执行。

### Q: 它能读多大文件？

单文件 ≤ 512 KiB 全量读取；超过此阈值自动切换到**大纲模式**（文件元信息 + 前 80 行 + 符号表 + 下一步操作建议）。超过 32 MiB 的文件直接拒绝（可用 `search_content` 按需搜索片段）。

### Q: 它能看到我的网络请求或数据库吗？

**不能。** Reasonix Code 的工具集是：文件系统 + Shell 命令 + Web 搜索。没有网络嗅探、数据库连接、浏览器 DOM 访问等能力。

### Q: 网络搜索的结果靠谱吗？

搜索结果来自你指定的搜索引擎（默认可选 Mojeek / SearXNG / Metaso / Tavily），助手会基于搜索结果回答，并引用来源链接。你可以通过 `/search-engine` 命令切换后端。

### Q: 为什么有时候它会先提交一个计划而不是直接改？

当改动涉及多个文件、有破坏性风险、或你的需求存在歧义时，它会主动提交计划让你把关。这是它的安全设计——**宁可多问一句，也不闷头改错**。

### Q: 如何处理跨会话记忆？

在对话中说"记住……"即可。下次开启新会话时，之前的记忆会自动加载到系统提示中。你可以通过 `/memory` 相关命令管理记忆。

### Q: 我想换项目怎么办？

目前不支持在同一个会话中切换工作目录。你需要退出当前会话（`Ctrl+C` 或关闭窗口），然后在新目录下重新启动：

```bash
cd ../other-project
reasonix code
```

### Q: 出现异常或工具调用失败怎么办？

- 如果 SEARCH 不匹配：助手会自动重新读取最新文件内容并重试
- 如果连续多次失败：系统会自动升级模型（flash → pro）并给出失败类型分析
- 如果完全卡住：回复 **"重试"** 或 **"换个方式"** 可以打断并重新开始

## 附录：术语表

| 术语 | 说明 |
|------|------|
| SEARCH/REPLACE | 文件编辑的精确匹配替换格式——先定位再替换，不允许模糊匹配 |
| Sandbox | 会话锁定的工作目录，所有文件操作不超出此边界 |
| Plan | 可审批的多步骤执行计划，含文件列表、风险评级 |
| Skill | 可复用的操作剧本，通过 `/skill` 或 `run_skill` 调用 |
| Subagent | 独立子会话，用于隔离复杂探索任务，不污染主会话上下文 |
| Multi_edit | 一次原子调用完成多个文件的编辑，任一失败则全回滚 |
| Memory | 持久化存储的偏好/决策，跨会话自动载入 |
